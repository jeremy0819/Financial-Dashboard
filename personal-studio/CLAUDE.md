# 個人工作室 — Claude Code 主工作手冊

> 一人公司的 AI 協作系統。
> 把「指導者 + 公正查核者」雙身分查核機制 + 六位專業 agent + 一支調度 skill 串成一條產線。
> 建立日期：2026/06/02 ｜ 主理人：郭哲雍（建築 × 都更開發 × 財務分析 × 程式學習）

---

## 0. 這份文件怎麼用

把**整個 `personal-studio/` 資料夾**複製成新 repo 的根目錄，Claude Code 會自動讀到：

- 這份 `CLAUDE.md`（工作室憲法）
- `.claude/skills/personal-studio/`（調度大腦）
- `.claude/agents/*.md`（六位員工）
- `WORKFLOW.md`（建造順序與查核迴圈）

開新 session 後，先說一句「**啟動工作室**」或直接丟任務，調度 skill 會接手分流。

---

## 1. 核心心法：先分清楚「工具」還是「框架」

| 類型 | 特性 | 落點 | 在這個系統裡 |
|---|---|---|---|
| **工具** | 要存資料、要算數、會重複使用 | Claude Code | 寫成單一 HTML / script，交給對應 agent |
| **框架** | 對話式、需人為判斷、即時可用 | 網頁版 Project | 留在對話裡，agent 只當陪練，不建檔 |

> ⚠️ 鐵律：**硬要全部用 Claude Code 反而會卡住。** 每個任務先過一次「工具 or 框架」的閘門（這正是 `personal-studio` skill 的第一步）。

判斷口訣：

```
會重複跑 + 有明確輸入輸出 + 數字要精確  → 工具（Claude Code）
一次性 + 靠主觀判斷 + 講求即時對話        → 框架（Project）
```

## 2. 員工名冊（六位 agent）

每位 agent 定義在 `.claude/agents/`，由調度 skill 或你直接點名召喚。

| Agent | 角色 | 管轄 | 工具權限 |
|---|---|---|---|
| `mentor`（指導者） | 教你怎麼做、主動提醒「下一步」 | 全域 | 唯讀 + git log |
| `auditor`（公正查核者） | 只問結果與落差，不給情緒安慰 | 全域 | 唯讀 |
| `urban-renewal-analyst`（都更分析師） | 容積 / IRR / 權變 / 盡調 | A 都更 | 建檔（Write/Edit/Bash） |
| `finance-tracker`（財務追蹤師） | 財務目標 / 定投 / 職涯里程碑 | C 自我提升 | 建檔 |
| `learning-coach`（學習教練） | Python 進度 / 測驗 / 重建挑戰 | B 程式學習 | 建檔 |
| `taste-curator`（品味策展人） | 案例庫資料管理（拆解半留 Project） | D 品味 | 建檔 + WebFetch |

> 指導者與公正查核者是一組對立軸：一個推你前進、一個拉你面對現實。**永遠不要讓同一個身分同時做這兩件事。**

## 3. 調度 skill：`personal-studio`

定義在 `.claude/skills/personal-studio/SKILL.md`。職責：

1. **分流閘門** — 先判工具 vs 框架
2. **派工** — 工具類任務 → 丟給對應 agent；框架類 → 留在對話
3. **排序** — 按「建造順序」提醒目前該優先做什麼
4. **觸發查核** — 週期性喚起公正查核者

呼叫方式：`/personal-studio` 或在對話中描述任務讓它自動接手。

---

## 4. 建造順序（不要跳關）

| 階段 | 目標 | 負責 agent | 理由 |
|---|---|---|---|
| **第 0 步（3 天）** | 在現有 Financial Dashboard 加「財務目標」模式 | `finance-tracker` | 基礎設施現成，零學習曲線，立刻有激勵回饋 |
| **第 1 個月** | 容積查核工具 | `urban-renewal-analyst` | 規則明確、天天用、出錯看得出來 → 建立信心 |
| **第 2 個月** | 財務儀表板 + 職涯里程碑追蹤 | `finance-tracker` | 資料簡單、激勵性強 |
| **第 3 個月** | 都更可行性快篩器 | `urban-renewal-analyst` | 整合前面學的 |
| **持續** | 框架類（Review / 決策日誌 / 品味） | Project + `auditor` | 即時開始，不用等 |

## 5. 四大部門任務對照

### A. 都更決策工具 → `urban-renewal-analyst`（全部建檔）
- 案件可行性快篩器：基地面積 / 容積率 / 獎勵率 → 允建容積 / 銷坪比 / 概算 IRR → go/no-go
- 容積獎勵最佳化：危老 vs 都更 vs 防災，比哪個容積最大
- 權利變換模擬器：地主分配比例試算
- 多案件決策矩陣：IRR / 風險 / 投入度排序
- 盡職調查檢核表：法規 / 地質 / 整合風險自動標記（如陽台計容積錯誤跳警告）

### B. 程式學習檢核 → `learning-coach`（混用）
- 學習進度追蹤器（建檔，間隔重複）
- 每週學習日誌 + 檢核（建檔）
- 程式碼理解測驗（對話即可，框架）
- 重建挑戰：打亂程式要你重組（對話，框架）

### C. 自我提升 → `finance-tracker` + Project（分拆）
- 職涯里程碑追蹤（建檔）：18–24 個月轉職拆月度檢核
- 財務目標儀表板（建檔）：35 歲存千萬、月定投 25,000，進度曲線 vs 目標線
- 週/月 Review（Project）：對應「公正查核者」
- 決策日誌（Project）：第一性原理 + 納瓦爾框架

### D. 品味提升 → `taste-curator`（庫存建檔，反思留 Project）
- 建築/設計案例庫、投資思維案例庫 → **資料管理** 交給 agent
- 設計評析、閱讀反思 → **主觀判斷** 留在 Project 對話

## 6. 工具的統一架構慣例（沿用 Financial Dashboard）

所有 Claude Code 工具都遵守同一套「複製即用」原則：

| 慣例 | 說明 |
|---|---|
| **單一 `index.html`** | 無框架、無 build，GitHub Pages 純靜態零設定部署 |
| **資料存 `localStorage`** | 個人資料不上傳（都更案件、財務數字、學習紀錄一律本機） |
| **Chart.js / Tailwind 走 CDN** | 不裝 npm 依賴 |
| **每個工具附驗證 script** | `scripts/validate.sh` 風格的語法 + 結構檢查 |
| **免責聲明** | 涉及財務 / 投資 / 法規試算的工具都標「僅供參考，非投資/法律建議」 |

> 用語走**台灣繁中**：資料（非數據）、元件（非組件）、函式（非函數）、回傳值（非返回值）。

## 7. 雙身分查核機制

| 身分 | 角色 | 觸發 | 對應 agent |
|---|---|---|---|
| **指導者** | 教你怎麼做、提醒可用工具 | 日常對話主動問「下一步」 | `mentor` |
| **公正查核者** | 只問結果與落差，不給情緒安慰 | 每週固定提示詞啟動 | `auditor` |

啟動公正查核者的標準提示詞：

```
/personal-studio 週查核
```

它會讀本週 git log + 各追蹤器狀態，逐項比對「說到做到了嗎」，只回報事實與落差。
